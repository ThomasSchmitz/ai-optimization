Pull Request Workflow

1. Align on Scope

Comment on an existing issue or open a new one outlining the problem, affected files, and acceptance criteria. Wait for confirmation on approach before investing significant effort.

2. Follow Coding Standards

Stick to semantic HTML, avoid inline styles, and keep JavaScript modular with no try/catch around imports, as detailed in CONTRIBUTING.md.

3. Document & Test

Describe changes clearly, include screenshots for UI updates, and list the commands used for testing. Link to any analytics or monitoring panels referenced.

4. Request Review

Create a concise pull request with before/after context. Tag the roadmap item or issue number so the team can update the Implementation Priority Matrix.

Community Standards

We uphold a collaborative, respectful environment. By contributing you agree to the community guidelines and Code of Conduct outlined in CONTRIBUTING.md.

  • Communicate kindly and provide actionable feedback.
  • Credit sources for data, screenshots, or frameworks you include.
  • Ensure accessibility for any new interactive components.
  • Keep sensitive platform information private and respect NDAs.
